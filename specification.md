# Формальная спецификация языка программирования Ferrous

## 1. Общее описание

Ferrous — процедурный компилируемый язык программирования со статической строгой номинальной типизацией. Сочетает простоту синтаксиса C с современными возможностями Rust: иммутабельность по умолчанию, вывод типов для локальных переменных, перегрузку функций. Предназначен для разработки приложений общего назначения под платформу x86-64.

**Основные характеристики:**
- Статическая, строгая, номинальная типизация.
- Отсутствие неявных преобразований типов — все приведения явные (`as`).
- Value semantics: все значения передаются и присваиваются по значению.
- Иммутабельность переменных по умолчанию, мутабельность через `mut`.
- Вывод типов для локальных переменных из контекста инициализации.
- Перегрузка функций по числу и типам параметров.
- Модульная организация: функции, структуры, пространства имён.
- Расширение файлов исходного кода: `.fer`.

**Структура программы:**
- Каждый исходный файл `.fer` является отдельным модулем.
- На верхнем уровне модуля допускаются только объявления: функций, структур, синонимов типов, пространств имён.
- Выражения и инструкции допускаются только внутри тела функций.
- Программа обязана содержать ровно одну функцию `main` — точку входа.

## 2. Лексическая структура

### 2.1 Пробельные символы и комментарии
- Пробельные символы: `' '`, `'\t'`, `'\n'`, `'\r'` игнорируются.
- Комментарии:
```
// Однострочный

/*
	Многострочный 
*/
```
Комментарии полностью отбрасываются лексическим анализатором и не передаются в поток токенов — они не влияют на семантику программы.

### 2.2 Идентификаторы
- Состоят из букв, цифр и символа `_`.
- Первый символ — буква или `_`.
- Примеры: `main`, `_value`, `counter123`.

### 2.3 Ключевые слова
```
let, mut, fn, return, if, else, while, break, continue,
struct, type, namespace, as, true, false, void
```
А также имена встроенных типов: `int8`, `int16`, `int32`, `int64`,
`uint8`, `uint16`, `uint32`, `uint64`, `float32`, `float64`, `bool`, `string`.

### 2.4 Литералы
- **Целые числа:** `123`, `42i64`, `255u8`, `0u32`
  - без суффикса — **нетипизированный целочисленный литерал** (см. §2.4.1)
  - с суффиксом — фиксированный тип (`i8`, `i16`, `i32`, `i64`, `u8`, `u16`, `u32`, `u64`)
- **Вещественные числа:** `3.14`, `2.5f32`, `0.0f64`
  - без суффикса — **нетипизированный вещественный литерал** (см. §2.4.1)
  - с суффиксом — фиксированный тип (`f32`, `f64`)
- **Булевы:** `true`, `false`
- **Строковые:** `"Hello, world!"`, `"line\n"`
  - экранирующие последовательности: `\n`, `\t`, `\r`, `\\`, `\"`, `\0`

#### 2.4.1 Нетипизированные числовые литералы
Числовой литерал без суффикса не получает фиксированный тип на этапе лексического анализа. Его тип определяется семантическим анализатором в точке использования по следующим правилам.

**Контексты, задающие ожидаемый тип:**
- аннотация типа переменной: `let x: T = <lit>`;
- параметр не-перегружённой функции: `f(<lit>)`, где существует единственная `fn f(x: T, ...)`;
- тип возвращаемого значения: `return <lit>;` внутри функции с типом возврата `T`;
- тип поля структуры в литерале: `Point { x: <lit> }`;
- тип элемента литерала массива: `[<lit>, ...]` при известном типе массива `[T; N]`;
- противоположный операнд бинарного оператора, если он имеет фиксированный числовой тип.

**Правила разрешения типа:**
1. Если в точке использования задан ожидаемый числовой тип и значение литерала укладывается в его диапазон — литерал получает этот тип.
2. Если ожидаемый тип не задан или не является числовым — применяется **тип по умолчанию**: `int32` для целых литералов, `float64` для вещественных.
3. Литерал с суффиксом имеет фиксированный тип и не подстраивается под контекст. Несоответствие суффикса ожидаемому типу — ошибка компиляции.
4. Нетипизированный литерал, значение которого не укладывается в ожидаемый тип — ошибка компиляции (`literal out of range for <T>`).

**Распространение ожидаемого типа через бинарные операторы:**
если один операнд — нетипизированный литерал, а другой имеет фиксированный числовой тип `T`, литерал принимает тип `T`. Если оба операнда — нетипизированные литералы, ожидаемый тип контекста применяется к обоим; при отсутствии контекста применяется тип по умолчанию.

**Примеры:**
```
let x: int8   = 5;              // 5 → int8 (из аннотации)
let y: uint32 = 0;              // 0 → uint32
let z: int64  = 100 + 200;      // оба литерала → int64
let a: int8   = 10;
let b: int8   = a + 1;          // 1 → int8 (другой операнд int8)
let k = 42;                     // контекста нет → int32 по умолчанию
let m = 42i64;                  // суффикс → int64
let err: int64 = 42i32;         // ОШИБКА: суффикс int32, ожидается int64
let big: int8 = 500;            // ОШИБКА: 500 не укладывается в int8
```

### 2.5 Разделители и операторы
| Разделители         | Операторы              |
| ------------------- | ---------------------- |
| `; , : { } [ ] ( )` | `+ - * / % =`          |
| `-> ::`             | `== != < <= > >=`      |
| `.`                 | `&& `||` `!`           |
|                     | `as` (приведение типа) |

## 3. Типы данных

### 3.1 Основные типы
| Тип        | Размер  | Описание                          |
| ---------- | ------- | --------------------------------- |
| `int8`     | 1 байт  | Знаковое целое, −128 … 127        |
| `int16`    | 2 байта | Знаковое целое                    |
| `int32`    | 4 байта | Знаковое целое (по умолчанию)     |
| `int64`    | 8 байт  | Знаковое целое                    |
| `uint8`    | 1 байт  | Беззнаковое целое, 0 … 255        |
| `uint16`   | 2 байта | Беззнаковое целое                 |
| `uint32`   | 4 байта | Беззнаковое целое                 |
| `uint64`   | 8 байт  | Беззнаковое целое                 |
| `float32`  | 4 байта | Вещественное, IEEE-754 single     |
| `float64`  | 8 байт  | Вещественное, IEEE-754 double     |
| `bool`     | 1 байт  | Булево значение (`true`/`false`)  |
| `string`   | —       | UTF-8 строка, неизменяемая        |
| `void`     | 0       | Отсутствие значения               |

### 3.2 Правила типизации
- **Номинальная типизация:** два типа совместимы только если их имена совпадают.
- **Без неявных приведений:** любое преобразование — через оператор `as`.
- **Value semantics:** все значения (включая массивы и структуры) копируются при присваивании и передаче в функцию.

### 3.3 Составные типы
```
// Структура
struct Point {
    x: float64,
    y: float64,
}

// Массив фиксированного размера (размер — часть типа)
let arr: [int32; 5] = [1, 2, 3, 4, 5];

// Синоним типа (прозрачен, не создаёт новый тип)
type Meters = int32;
```

### 3.4 Преобразования типов
- **Неявные преобразования отсутствуют.**
- **Явные (через `as`):**
```
let a: int32 = 10;
let b: int64 = a as int64;   // widening
let c: int8  = a as int8;    // narrowing (по модулю 2^8)
let f: float64 = a as float64;
let i: int32  = 3.7 as int32; // усечение к нулю → 3
```
- Разрешены между числовыми типами и между `bool` → число (`true` → 1, `false` → 0).
- Оператор `as` не определён для `string`, структур и массивов.
### 3.5 Поведение при переполнении
- Целочисленное переполнение — wrapping (по модулю 2ⁿ).
- Вещественное — по правилам IEEE-754 (NaN, ±inf).

## 4. Функции

### 4.1 Определение
```
fn function_name(param1: Type1, param2: Type2) -> ReturnType {
    /* тело функции */
}
```
Если функция не возвращает значения, указывается `-> void` или стрелка опускается.

### 4.2 Вызов
```
let result: ReturnType = function_name(arg1, arg2);
```

### 4.3 Перегрузка
Функции могут иметь одинаковое имя при различии числа или типов параметров (но не только типа возврата):
```
fn add(a: int32, b: int32) -> int32 { 
	return a + b; 
}

fn add(a: float64, b: float64) -> float64 {
	return a + b; 
}

fn add(a: int32, b: int32, c: int32) -> int32 {
	return a + b + c;
}
```
Разрешение перегрузки требует **точного совпадения** типов аргументов. Неявное ранжирование кандидатов не производится.

Нетипизированные числовые литералы (§2.4.1) не участвуют в выборе перегрузки: каждая кандидатная перегрузка сопоставляется только по аргументам, имеющим фиксированный тип. Если после этого остаётся более одной подходящей перегрузки — программа отклоняется с ошибкой `ambiguous call`. Для устранения неоднозначности необходимо зафиксировать тип литерала — суффиксом или явным приведением через `as`:
```
fn add(a: int32, b: int32) -> int32 { return a + b; }
fn add(a: int64, b: int64) -> int64 { return a + b; }

add(10, 20);           // ОШИБКА: ambiguous call — подходят обе перегрузки
add(10i64, 20i64);     // OK: выбрана int64-версия
add(10 as int32, 20);  // OK: первый аргумент фиксирует int32, второй — нетипизированный,
                       // единственная подходящая — int32-версия, второй литерал → int32
```

### 4.4 Точка входа
Программа обязана содержать функцию `main`:
```
fn main() -> int32 {
    return 0;
}
```
Возвращаемое значение передаётся как код завершения процесса.

### 4.5 Семантика вызова
- Call-by-value: все аргументы копируются при передаче.
- Порядок вычисления аргументов — слева направо.

## 5. Переменные и мутабельность

### 5.1 Объявление
```
let x: int32 = 10;        // иммутабельная
let mut y: int32 = 20;    // мутабельная
let z = 3.14;             // тип выведен как float64
```

### 5.2 Правила
- Переменная обязана быть инициализирована при объявлении.
- Повторное присваивание иммутабельной переменной — ошибка компиляции.
- Shadowing разрешён: внутреннее объявление перекрывает внешнее.
- Двойное объявление в одной области видимости — ошибка.

### 5.3 Вывод типов
Если тип в `let` опущен, он выводится из выражения справа:
```
let x = 10;               // int32
let y = 3.14;             // float64
let s = "hello";          // string
let arr = [1, 2, 3];      // [int32; 3]
```
Вывод только локальный: параметры функций и возвращаемые типы указываются явно.

## 6. Выражения и операторы

### 6.1 Виды выражений
Язык поддерживает следующие виды выражений:

- **Литералы:** целые, вещественные, булевы, строковые.
- **Литералы массива:** `[1, 2, 3]` — все элементы должны иметь одинаковый тип.
- **Литералы структуры:** `Point { x: 1.0, y: 2.0 }` — все поля обязаны быть инициализированы.
- **Имена:** простые (`x`) и квалифицированные (`math::square`).
- **Унарные:** `-x` (арифметическое отрицание), `!x` (логическое отрицание).
- **Арифметические:** `+`, `-`, `*`, `/`, `%`.
- **Сравнения:** `==`, `!=`, `<`, `<=`, `>`, `>=`.
- **Логические:** `&&`, `||`, `!`.
- **Приведение типа:** `expr as Type`.
- **Вызов функции:** `f(arg1, arg2)`.
- **Индексирование массива:** `arr[i]` — `i` должен иметь целочисленный тип.
- **Доступ к полю структуры:** `p.x`.
- **Доступ к элементу пространства имён:** `math::square`.
- **Скобки для группировки:** `(expr)`.

### 6.2 Приоритет и ассоциативность
| Приоритет | Операторы                   | Ассоциативность |
| --------- | --------------------------- | --------------- |
| 1         | `()` `[]` `.` `::`          | слева-направо   |
| 2         | унарные `-` `!`             | справа-налево   |
| 3         | `as`                        | слева-направо   |
| 4         | `* / %`                     | слева-направо   |
| 5         | `+ -`                       | слева-направо   |
| 6         | `< <= > >=`                 | слева-направо   |
| 7         | `== !=`                     | слева-направо   |
| 8         | `&&`                        | слева-направо   |
| 9         | `\|\|`                      | слева-направо   |
| 10        | `=` (присваивание)          | справа-налево   |

### 6.3 Семантика операторов
- **Унарный `-`** определён для знаковых целых и вещественных типов.
- **Унарный `!`** определён только для `bool`.
- **Арифметические `+ - * / %`** требуют операндов одного числового типа, результат — того же типа. Операция `+` разрешена для строк ввиде конкатенации
- **Операторы сравнения `< <= > >=`** определены для числовых типов, результат — `bool`.
- **`== !=`** определены для всех встроенных типов и структур (поэлементное сравнение полей); результат — `bool`.
- **`&& ||`** определены только для `bool`, используют short-circuit (ленивую оценку): правый операнд не вычисляется, если результат определён левым.
- Деление целых — усечение к нулю (`-7 / 2 == -3`).
- Деление на ноль (для целых) — ошибка времени выполнения.

### 6.4 Порядок вычисления
- Операнды бинарных операторов вычисляются **слева направо**.
- Аргументы функций вычисляются **слева направо**.
- Индекс массива `arr[i]` — сначала вычисляется `arr`, затем `i`.
- Исключение: операторы `&&` и `||` используют ленивую оценку (правый операнд может не вычисляться).

### 6.5 Строковые операции
- Конкатенация: `s1 + s2`
- Длина: встроенная функция `len(s)` → `int32`
- Сравнение: `==`, `!=`

## 7. Управляющие конструкции

### 7.1 Условный оператор
```
if condition {
    // ...
} else if other_condition {
    // ...
} else {
    // ...
}
```
Условие обязано иметь тип `bool` — никаких «truthy» значений.

### 7.2 Циклы
```
while condition {
    // ...
}
```
Цикл `while` — единственная базовая конструкция цикла.

### 7.3 Прерывания
- `break` — немедленный выход из ближайшего цикла.
- `continue` — переход к следующей итерации.
- `return expr;` — возврат значения из функции.
- `return;` — возврат из функции с типом `void`.

Все пути функции с ненулевым типом возврата обязаны завершаться `return`.

### 7.4 Блок и нулевая инструкция
- Блок `{ ... }` вводит новую область видимости.
- Нулевая инструкция `;` не производит эффекта.

## 8. Массивы и строки

### 8.1 Массивы
```
let arr: [int32; 5] = [1, 2, 3, 4, 5];
let first: int32 = arr[0];
```
- Размер является частью типа: `[int32; 5]` и `[int32; 10]` — разные типы.
- Индексация с нуля.
- Выход за границы — ошибка времени выполнения.
- Копируются целиком при присваивании и передаче в функцию.

### 8.2 Строки
```
let s: string = "Hello";
let s2: string = s + ", world!";
let n: int32 = len(s2);
```
- UTF-8 кодировка.
- **Модель хранения:** указатель на байтовый буфер + длина в байтах (без завершающего нуля). Это позволяет хранить любые UTF-8 последовательности и получать длину за O(1).
- `len(s)` возвращает длину в **байтах**, не в символах.
- Неизменяемы: после создания модифицировать нельзя.
- Value semantics: копируются при присваивании (копируется содержимое буфера, не только указатель).

## 9. Структуры и пространства имён

### 9.1 Структуры
```
struct Point {
    x: float64,
    y: float64,
}

let p: Point = Point { x: 1.0, y: 2.0 };
let px: float64 = p.x;
```
- Все поля обязаны быть инициализированы в литерале.
- Доступ через точку: `p.x`.
- Мутабельность полей следует из мутабельности переменной.

### 9.2 Синонимы типов
```
type Meters = int32;
let distance: Meters = 100;  // полностью эквивалентно int32
```

### 9.3 Пространства имён
```
namespace math {
    fn square(n: int32) -> int32 {
        return n * n;
    }
}

let s: int32 = math::square(5);
```
- Могут быть вложенными.
- Доступ через оператор `::`.

---

## 10. Область видимости и разрешение имён

- Лексическая (статическая) область видимости.
- Идентификатор разрешается в ближайшей охватывающей области.
- Shadowing разрешён, двойное объявление в одной области — ошибка.
- Использование до объявления не допускается (кроме функций и типов верхнего уровня, видимых во всём файле).
- Переменная существует от точки объявления до конца блока.

## 11. Встроенные функции

| Функция   | Сигнатура                       | Назначение                            |
| --------- | ------------------------------- | ------------------------------------- |
| `print`   | перегружена, см. §11.1          | Вывод значения в stdout               |
| `println` | перегружена, см. §11.1          | Вывод значения в stdout с `\n` в конце|
| `input`   | `fn input() -> string`          | Чтение строки из stdin                |
| `len`     | `fn len(s: string) -> int32`    | Длина строки в байтах                 |
| `exit`    | `fn exit(code: int32) -> void`  | Завершение с кодом возврата           |
| `panic`   | `fn panic(msg: string) -> void` | Аварийное завершение с сообщением     |

### 11.1 Перегрузки `print` и `println`

Функции `print` и `println` перегружены для всех встроенных примитивных типов. Для каждого из приведённых ниже типов существует по одной перегрузке `print(x: T) -> void` и `println(x: T) -> void`:

- строковый: `string`
- булев: `bool` (выводится как `true` / `false`)
- знаковые целые: `int8`, `int16`, `int32`, `int64` (десятичная запись со знаком)
- беззнаковые целые: `uint8`, `uint16`, `uint32`, `uint64` (десятичная запись без знака)
- вещественные: `float32`, `float64` (десятичная запись; специальные значения — `nan`, `inf`, `-inf`)

`println` дополнительно выводит символ `\n` после значения. `print` никаких разделителей не добавляет.

**Составные типы** (структуры, массивы) **не поддерживаются**: их поля/элементы нужно выводить по отдельности либо предварительно преобразовывать в `string`.

**Разрешение перегрузки с литералами.** В силу §4.3 нетипизированные числовые литералы не участвуют в выборе перегрузки. Поскольку `print` и `println` имеют несколько числовых перегрузок, вызов с одним нетипизированным литералом без контекста — ошибка `ambiguous call`:
```
print(42);            // ОШИБКА: ambiguous call — подходят все целочисленные перегрузки
print(42i32);         // OK: суффикс фиксирует int32
print(42 as int32);   // OK: явное приведение фиксирует int32
print(3.14f64);       // OK: суффикс фиксирует float64
print("hello");       // OK: строковый литерал однозначно string
print(true);          // OK: булев литерал однозначно bool

let x: int32 = 10;
print(x);             // OK: x имеет фиксированный тип int32
```

Если нужно вывести сразу несколько значений, делается последовательность вызовов:
```
print("sum = "); println(sum);
```

## 12. Ошибки времени выполнения

При возникновении ошибки программа завершается с сообщением вида:
```
runtime error: <сообщение> at line <N>
```
и ненулевым кодом возврата (по умолчанию 1).

Обязательно обрабатываемые ошибки:

| Ситуация                  | Сообщение                                        |
| ------------------------- | ------------------------------------------------ |
| Деление на ноль           | `runtime error: division by zero at line N`      |
| Остаток от деления на ноль| `runtime error: modulo by zero at line N`        |
| Выход за границы массива  | `runtime error: index out of bounds at line N`   |
| Вызов `panic(msg)`        | `runtime error: <msg> at line N`                 |

Сообщения выводятся в стандартный поток ошибок (`stderr`). Номер строки соответствует позиции операции в исходном коде.

## 13. Примеры программ

### 13.1 Hello World
```
fn main() -> int32 {
    println("Hello, World!");
    return 0;
}
```

### 13.2 Сумма элементов массива
```
fn main() -> int32 {
    let arr: [int32; 5] = [1, 2, 3, 4, 5];
    let mut sum: int32 = 0;
    let mut i: int32 = 0;
    while i < 5 {
        sum = sum + arr[i];
        i = i + 1;
    }
    print(sum);
    return 0;
}
```

### 13.3 Структуры и функции
```
struct Point {
    x: float64,
    y: float64,
}

fn distance_sq(a: Point, b: Point) -> float64 {
    let dx = a.x - b.x;
    let dy = a.y - b.y;
    return dx * dx + dy * dy;
}

fn main() -> int32 {
    let p1 = Point { x: 0.0, y: 0.0 };
    let p2 = Point { x: 3.0, y: 4.0 };
    print(distance_sq(p1, p2));
    return 0;
}
```

### 13.4 Перегрузка функций
```
fn max(a: int32, b: int32) -> int32 {
    if a > b { return a; } else { return b; }
}

fn max(a: float64, b: float64) -> float64 {
    if a > b { return a; } else { return b; }
}

fn main() -> int32 {
    let i = max(10, 20);
    let f = max(1.5, 2.5);
    return 0;
}
```

### 13.5 Пространства имён
```
namespace math {
    fn square(n: int32) -> int32 {
        return n * n;
    }

    fn cube(n: int32) -> int32 {
        return n * square(n);
    }
}

fn main() -> int32 {
    let result = math::cube(3);
    println(result);  // 27
    return 0;
}
```
